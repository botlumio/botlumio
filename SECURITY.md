# Security Policy

Thank you for helping keep **Lumiobot** and our users safe.

## Supported Versions

We currently provide security updates for the following versions:

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

**Please do not open public GitHub issues for security vulnerabilities.**

If you discover a security issue, report it privately so we can fix it before
it is publicly disclosed.

Preferred channels (in order):

1. **GitHub Security Advisories** — open a private report at
   <https://github.com/botlumio/botlumio/security/advisories/new>
2. **Direct message** to the founder on X: [@Zhenya718](https://x.com/Zhenya718)
3. **Telegram**: contact the bot owner via [@ailumio_bot](https://t.me/ailumio_bot)

When reporting, please include:

- A clear description of the vulnerability
- Steps to reproduce (proof-of-concept if possible)
- The potential impact
- Any suggested mitigation

## Response Timeline

| Stage                  | Target                  |
| ---------------------- | ----------------------- |
| Initial acknowledgment | within **48 hours**     |
| Triage & severity      | within **5 days**       |
| Fix or mitigation      | within **30 days**      |
| Public disclosure      | coordinated with reporter |

We follow a **coordinated disclosure** model. Once a fix is released, we will
credit the reporter in the release notes (unless anonymity is requested).

## Scope

In scope:

- The Lumiobot Telegram bot ([@ailumio_bot](https://t.me/ailumio_bot))
- The website at [lumiobot.fun](https://lumiobot.fun)
- This GitHub repository (code, CI, configs)

Out of scope:

- Third-party services we depend on (Anthropic, Jina AI, Telegram, Railway, etc.)
- Social engineering of staff or users
- Denial of service that requires excessive traffic
- Automated scanner output without a working proof-of-concept

## Safe Harbor

We will not pursue legal action against researchers who:

- Make a good-faith effort to comply with this policy
- Avoid privacy violations, data destruction, and service disruption
- Give us reasonable time to respond before any public disclosure

## Secrets & Credentials

If you accidentally find leaked credentials (API keys, tokens, wallets) in
this repository or any deployment, **report immediately** via the channels
above and **do not** use them.
